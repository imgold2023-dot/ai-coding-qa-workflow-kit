# Preview: PRD to Playwright QA Starter Pack

The full pack is a sequenced AI workflow for turning rough feature ideas into QA-ready product work.

## Sequence

```text
rough feature idea
  -> PRD clarification
  -> PRD draft
  -> acceptance criteria
  -> risk discovery
  -> test case matrix
  -> test data policy
  -> Playwright automation plan
  -> spec skeleton
  -> test code review
  -> release smoke checklist
```

## Included

- 10 sequenced prompts
- 5 reusable templates
- 1 fictional end-to-end SaaS feature example

## Example Prompt Excerpt

```text
You are a senior QA engineer specializing in risk-based web app testing.

Review the PRD and acceptance criteria below. Identify the highest-value test risks before generating a full test matrix.

Rules:
- Focus on user impact, data integrity, permission mistakes, and release risk.
- Separate likely risks from speculative risks.
- Identify what should be automated and what should remain manual.
- Avoid requiring real customer data, production credentials, cookies, payment accounts, or sensitive information.
- Prefer dummy data and sandbox conditions.
```

## Who It Is For

- indie developers shipping web features
- QA engineers turning manual cases into automation plans
- technical founders using AI coding tools
- small teams using ChatGPT, Claude, Cursor, Codex, or Playwright

## Safety Boundary

Use dummy data first. Do not paste real customer data, production tokens, cookies, browser profiles, bank details, payment account credentials, or sensitive information into any AI tool.
